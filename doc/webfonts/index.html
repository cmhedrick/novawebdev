Fun with fonts

The knowledge here was gleaned from CSS Tricks: Using @font-face, among other sources.

After converting your font to multiple formats, add to your CSS, creating a name of your choosing for the font-family:

      @font-face {
          font-family: "cc_iconsregular";
          src: url("cc-icons-webfont.eot");
          src: url("cc-icons-webfont.eot?#iefix")          format("embedded-opentype"),
               url("cc-icons-webfont.woff2")               format("woff2"),
               url("cc-icons-webfont.woff")                format("woff"),
               url("cc-icons-webfont.ttf")                 format("truetype"),
               url("cc-icons-webfont.svg#cc_iconsregular") format("svg");
          font-weight: normal;
          font-style: normal;
      }

It's relatively self-explanatory:

  1. Enclose in @font-face { … }.
  2. Choose a font-family name.
  3. Provide a src URL to the font file.
  4. Optionally, provide fallback src URLs along with a format("…") for each corresponding src, separated by commas.
  5. The font-weight and font-style attributes don't seem necessary to me, but they don't hurt.

Then just add <span style="font-family:cc_iconsregular;">a</span> — or better yet, implement a class and use it — wherever you need a a as a character, rather than an inline image. The Creative Commons Share-Alike (SA) glyph is mapped to lower-case "a" (a.k.a. &#97;).

Obviously, you'll need some sort of glyph table in order to know the mapping between the Unicode value to the glyph that will be shown.

Also, less obviously, the font file(s) must reside on the same site as the CSS. The url referenced in the style sheet cannot reference a font file on another system.

Unfortunately, in this particular case, the glyph looks like crap at anything less than a font size of 24px, which makes it hard to use inline without making the rest of the text a bit large for my tastes. Particularly, in the footer where I would usually want SMALLER than normal text.

The web-based conversion service Font Squirrel (http://www.fontsquirrel.com), mentioned in the CSS Tricks article, produces a nice demo page in addition to converting the fonts. The demo, downloaded as part of the generated ZIP file, includes a glyph table to help you map the Unicode value to the glyph, as well as a sample layout, and more detailed instructions on how to include them in a web page.

(The other conversion service mentioned in the article isn't nearly as helpful.)
